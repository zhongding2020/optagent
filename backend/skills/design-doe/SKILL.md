# 试验设计 (DOE)

## 核心规则
- **严禁**生成或执行 Python/代码来生成设计矩阵。通过对话引导用户确定DOE方案。
- 完成本步骤后务必调用 `step_complete` 工具，提供结构化的试验设计摘要。

## Steps

### 1. 选择DOE类型
- **全因子设计**：2^k 次试验，适合因子数 ≤ 5
- **部分因子设计**：减少试验次数，适合因子数较多时的初筛
- **响应面设计 (RSM)**：中心复合设计或Box-Behnken，适合寻找最优区域
- **田口设计**：关注稳健性，适合噪声因子分析
- 根据因子数和资源限制推荐最合适的方案

### 2. 确定试验矩阵
- 因子数与水平数
- 试验次数估计
- 随机化顺序
- 是否需要分区组（消除噪声）

### 3. 试验资源规划
- 每次试验的时间和成本估算
- 是否需要重复试验
- 人员分工和设备安排

### 4. 输出试验方案
- 调用 `step_complete` 工具，输出格式：
```
{
  "doe_type": "全因子/部分因子/响应面/田口",
  "factors": [{"name": "参数名", "levels": [水平值列表]}],
  "total_runs": 试验次数,
  "replicates": 重复次数,
  "randomized": true/false,
  "estimated_cost": "成本估算描述"
}
```
